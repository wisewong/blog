curl命令踩的坑

使用curl命令执行get请求，带多个参数： 

curl localhost:8080/user/binding/query?userId=123456&wrapperId=123&from=test
[1] 8937
[2] 8938
{"ver":"1.0.0","status":1,"message":"Required String parameter 'wrapperId' is not present","data":null,"ok":false}
[1]-  Done                    curl localhost:8080/user/binding/query?userId=123456
[2]+  Done                    wrapperId=123

返回结果提示缺少参数wrapperId，但是上述url中的确有wrapperId。

看最后两行的结果，发现是url被&号截开，在shell中被当做两条命令来执行的。

解决方案

在url外面加上引号： 

curl "localhost:8080/user/binding/query?userId=123456&wrapperId=123&from=test"
